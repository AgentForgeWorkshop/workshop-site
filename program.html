Hands-on tutorials

From a single ReAct loop to an autonomous assistant.

Each tutorial pairs a 30-minute talk from a domain expert with guided, coding-level hands-on work. All notebooks are prepared in advance on a dedicated GitHub repository.

1Tutorial

From Chatbot to Agent

The core difference between a standard LLM and an AI agent, explored through the ReAct (Reason + Act) framework. We build a literature-scout agent that calls an arXiv search tool to demonstrate the loop in practice.

Agent vs. LLMReAct frameworkMemory + toolsLiterature agent
2Tutorial

Agentic Memory

Scientific research demands deep context. This tutorial dives into memory and RAG architectures — giving agents access to domain-specific knowledge bases without hallucinating facts — plus memory compression for limited context windows.

Markdown · SQL · vector · graphAgentic RAGMemory compression
3Tutorial

MCP & Skills

An assistant is only as good as the tools it can use. Tool-calling and the Model Context Protocol — the “USB-C for AI agents” — plus more capable agents on platforms like Claude Code and Antigravity, alongside scientific tools such as ToolUniverse.

Tool schemasMCP client–serverFile manipulationChaining skills
4Tutorial

Multi-Agent Orchestration

Complex scientific workflows need specialised roles. Orchestrate multiple agents that communicate, debate, and delegate — across sequential pipelines, manager–worker hierarchies, and collaborative-debate architectures.

Role specializationStatePipeline · hierarchical · debate
5Tutorial

Autonomous Agents

An end-to-end research assistant that integrates everything from the previous sessions into one cohesive system. We introduce always-on, system-level frameworks such as OpenClaw and NanoClaw — with an optional hands-on setup on a remote VPS.

End-to-end assistantOpenClaw · NanoClawOptional VPS lab

Driving domain-specific innovation

The hackathon.

Form working groups of 3–5 aligned by scientific discipline — bioinformatics, materials science, physics. Bring your own tools, datasets, and research bottlenecks, then collaboratively design, prompt, and deploy a custom agent.

Judged on

Domain impact Technical execution Popular vote

What each team ships

  • A five-minute live demo
    The agent executing a real domain-specific task, with a 1–2 slide architecture overview of its tools and prompts.
  • A shared repository
    Code plus examples of successful and failed reasoning traces.
  • A 1–2 page report
    Insights, challenges, and open questions from building agents for scientific tasks.
